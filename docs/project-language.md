# Project Language Guard

This document establishes the official taxonomy and phrasing guidelines for the Survival Operating System (SOS) project. All developers, documentation updates, and AI prompts must adhere strictly to these terminology rules.

## 1. Project Framing
*   **Core Identity**: SOS is a **local-first, offline survival command center application** and **homestead management dashboard**.
*   **Branding**: "SurvivalOS" and "SOS" are product names representing the local application interface and command center dashboard experience. They do **not** refer to a machine operating system.
*   **System Architecture**: SOS runs entirely as a standard web dashboard environment (Node/Express backend, Vite/React frontend) on a local machine.

## 2. Forbidden Terminology
*   **Do NOT** use machine-level operating system migration language.
*   **Do NOT** include terms relating to:
    *   Bootable drives / bootloaders
    *   BIOS configurations
    *   UEFI configurations
    *   ISO installation/flashing (except when referring to mounting pre-existing CD3WD reference ISO files inside Windows File Explorer)
    *   Linux appliance migration / bare-metal hypervisors
*   **Do NOT** suggest migrating the active application away from its standard Node/React offline client configuration.

## 3. Allowed Terminology
*   **Logistical Dashboard**: "offline command center app", "local survival dashboard", "homestead management database".
*   **Knowledge Base**: "offline reference library", "neural document search", "R.A.N.G.E.R. (Resource And Neural Guidance Engine Responder)-powered local knowledge system".
*   **Task Runners**: "sync crawler", "metadata decoder", "vision OCR library script".
